# Data Flow - Slider Feature

## Flow 1: Public Slider Listing

```
Client
  |
  GET /api/v1/general/sliders?limit=5
  |
  v
Route: routes/api.php → General\SliderController@index
  |
  v
SliderService::getSliders($request)
  |
  +-- Applies HasChannelFilter trait
  |     |-- Filters products by home channel if applicable
  |
  +-- Applies filters from request:
  |     |-- start_date / end_date → whereBetween('created_at', ...)
  |     |-- slidersId → whereIn('id', ...)
  |     |-- limit → take($limit)
  |
  +-- Orders results:
  |     |-- orderBy('order', $request->order ?? 'asc')
  |
  +-- Only active sliders:
        |-- Slider::active()->get()
  |
  v
SliderResource collection (App\Http\Resources\Slider)
  |
  Maps each slider to:
    - id
    - title (translated)
    - slug
    - status
    - image { desktop URL, mobile URL }
    - products (when loaded)
  |
  v
JSON Response
{
  "data": [
    {
      "id": 1,
      "title": "Summer Sale",
      "slug": "summer-sale",
      "status": true,
      "image": { "desktop": "...", "mobile": "..." },
      "products": [...]
    }
  ]
}
```

## Flow 2: Public Slider Detail by Slug

```
Client
  |
  GET /api/v1/general/sliders/summer-sale
  |
  v
General\SliderController@getSliderBySlug('summer-sale')
  |
  v
SliderService::getSliderBySlug('summer-sale')
  |
  +-- Slider::active()
  |     ->where('slug', 'summer-sale')
  |     ->with(['products' => function ($q) {
  |         $q->applyChannelHomeFilter();           // Channel filter
  |     }])
  |     ->firstOrFail()
  |
  +-- ProductService::enrichCollectionWithPricing(
        $slider->products, $request
      )                                             // Adds pricing data
  |
  v
SliderResource (single)
  |
  JSON Response
```

## Flow 3: Admin Slider Creation

```
Client
  |
  POST /api/v1/sliders
  Authorization: Bearer <token>
  Content-Type: multipart/form-data
  Body: title[en]=Summer Sale, image_desktop=<file>, image_mobile=<file>, ...
  |
  v
Middleware: sanctum (auth)
  |
  v
Controller constructor middleware: permission:create-slider
  |
  v
SliderController@store(SliderCreateRequest $request)
  |
  +-- Request validation:
  |     |-- title.* (required|array)
  |     |-- title.en (required|string|unique_translation)
  |     |-- title.ar (required|string|unique_translation)
  |     |-- image_desktop (required|image|mimes|max:2048)
  |     |-- image_mobile (required|image|mimes|max:2048)
  |     |-- status (sometimes|in:1,0)
  |     |-- products (sometimes|array)
  |     |-- products.* (exists:products,id)
  |
  v
SliderRepository::createSlider($request)
  |
  +-- DB::beginTransaction()
  |
  +-- Create Slider model:
  |     |-- $slider = Slider::create([
  |     |     'title' => $request->title,    // translatable array
  |     |     'status' => $request->status ?? 0
  |     |   ])
  |     |-- Slug auto-generated by model boot:
  |     |   $slider->slug = Str::slug($request->title['en'])
  |     |-- Order auto-assigned by Sortable trait
  |
  +-- Upload images:
  |     |-- $slider->addMedia($request->file('image_desktop'))
  |     |       ->toMediaCollection('slider-image-desktop')
  |     |-- $slider->addMedia($request->file('image_mobile'))
  |     |       ->toMediaCollection('slider-image-mobile')
  |
  +-- Sync products:
  |     |-- if $request->products:
  |     |     $slider->products()->sync($request->products)
  |
  +-- DB::commit()
  |
  v
Admin SliderResource response (201)
{
  "data": {
    "id": 11,
    "title": { "en": "Summer Sale" },
    "slug": "summer-sale",
    "status": false,
    "order": 6,
    "image": { "desktop": "...", "mobile": "..." },
    "products": []
  }
}
```

## Flow 4: Admin Slider Update (Image Replacement)

```
Client
  |
  PUT /api/v1/sliders/11
  Body: image_desktop=<new-file>, title[en]="New Title"
  |
  v
SliderRepository::updateSlider($request, 11)
  |
  +-- DB::beginTransaction()
  |
  +-- Update model:
  |     |-- $slider->update($data)
  |     |-- Slug auto-regenerated if English title changed
  |
  +-- Replace images (if new files provided):
  |     |-- $this->updateSingleImage(
  |     |     $slider, $request->file('image_desktop'),
  |     |     'sliders-desktop'     // Note: different collection name!
  |     |   )
  |     |-- $this->updateSingleImage(
  |     |     $slider, $request->file('image_mobile'),
  |     |     'sliders-mobile'      // Note: different collection name!
  |     |   )
  |
  +-- Resync products:
  |     |-- $slider->products()->sync($request->products ?? [])
  |
  +-- DB::commit()
  |
  v
Updated SliderResource response
```

## Flow 5: Slider Status Toggle

```
Client
  |
  PATCH /api/v1/sliders/change-status
  Body: { "id": 5 }
  |
  v
SliderController@changeStatus(Request $request)
  |
  +-- Validate: $request->has('id')
  |
  v
SliderRepository::changeStatus(5)
  |
  +-- $slider = Slider::findOrFail(5)
  +-- $slider->update(['status' => !$slider->status])  // Toggle
  |
  v
Response: { "message": "Slider status changed successfully" }
```

## Flow 6: Slider Reorder

```
Client
  |
  PUT /api/v1/sliders/reorder
  Body: { "sliders": [3, 1, 5, 2, 4] }
  |
  v
SliderController@reorder(Request $request)
  |
  +-- Validate: $request->has('sliders') && is_array
  |
  v
SliderRepository::reorder([3, 1, 5, 2, 4])
  |
  +-- $this->setNewOrder([3, 1, 5, 2, 4])
  |     |-- Spatie Sortable trait
  |     |-- Updates 'order' column for each ID:
  |     |   3 → order=1, 1 → order=2, 5 → order=3, 2 → order=4, 4 → order=5
  |
  v
Response: { "message": "Sliders reordered successfully" }
```
